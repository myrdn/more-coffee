Using Tailwind CSS with Zola

This post is written listening to Népal.

Our website at clic2000 is pure handcoding in HTML/SASS/JS, no CMS, no static site generator. We like it, it is easy to understand, secure and fast. We just need to pull the content on our server and compile the SASS files et voilà. However we have to admit that editing HTML while adding content is not ideal. In recent discussions we decided to give a try to static site generators.

In a near future we would like to add a blog to our website. Also one redondant task is to had words about our recents works on our references page. Keeping editing HTML and managing files would be painfull and time consuming. We use markdown on a daily basis on our notes, our internal doc and GitHub, using it for our website would be a real plus. We also like the idea to separate structure from content.

Which SSG should we use ? Mathilde uses it for her personal homepage and this article made us decide to use Zola. It is written in Rust, it comes in a single binary and has nice features like Sass compilation or syntax highlighting. It uses Tera template engine, which I haven't tried to the fullest but the include feature is something we miss in our actual website.

Before working on clic2000 website I decided to build this little piece of internet and this is how I managed configuring Zola using Tailwind CSS. It is based on this great post except it uses postcss-cli instead of parcel to compile the styles of the website.

You need Zola and npm installed on your computer and you are ready to go.

Setup

Create the directory structure of Zola :

zola init

Create package.json file and install packages : 

npm init
npm install tailwindcss postcss postcss-cli autoprefixer

Create styles folder and styles.css file :

mkdir styles
cd styles
touch styles.css

Add this content to your styles.css file :

@tailwind base;
@tailwind components;
@tailwind utilities;

Configure PostCSS to use Tailwind plugin. Add this content at the root of the project in postcss.config.js file :

module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  }
}

Now configure PostCSS to compile your CSS files. In package.json : 

"scripts": {
  "watch:css": "postcss styles/styles.css -o static/styles.css -w",
  "build:css": "postcss styles/styles.css -o static/style.css"
}

Templates

Add the following to your templates : 

{% block css %}
	      <link rel="stylesheet" href="{{ get_url(path="styles.css", trailing_slash=false) | safe }}">
{% endblock css %}

Run

You can now work on your theme, just run zola serve and npm run watch:css in two terminal tabs.

Run npm run build:css before production.
